  <action>
    Create src/soma/reflexes.py as a pure function module (per D-05):

    1. Define ReflexResult frozen dataclass (per research Pattern 1):
       - allow: bool, reflex_kind: str = "", block_message: str | None = None, inject_message: str | None = None, detail: str = ""

    2. Define constants:
       - BLOCKING_REFLEXES = {"blind_edits", "bash_failures", "thrashing", "retry_dedup"} (per D-09 through D-12)
       - INJECTION_REFLEXES = {"error_rate", "research_stall", "agent_spam"} (per D-13, D-14)

    3. Implement evaluate() function with signature:
       ```python
       def evaluate(
           tool_name: str, tool_input: dict, action_log: list[dict],
           pressure: float = 0.0, config: dict | None = None,
           bash_history: list[str] | None = None, workflow_mode: str = "",
       ) -> ReflexResult:
       ```

    4. Evaluate flow:
       a. Check override first: if config.get("override_allowed") and "SOMA override" in tool_input.get("command", "") -> return allow=True (per D-27)
       b. Check retry_dedup before patterns (needs tool_input, not in patterns.py): normalize Bash command via " ".join(cmd.split()), check against bash_history list of normalized commands (per D-10)
       c. Call patterns.analyze(action_log, workflow_mode) to get active patterns
       d. For each pattern in results:
          - If kind in BLOCKING_REFLEXES and config.get(kind, True): check _should_block(pattern, tool_name, tool_input) and return allow=False with formatted block_message
          - If kind in INJECTION_REFLEXES and config.get(kind, True): return allow=True with inject_message
       e. Default: return ReflexResult(allow=True)

    5. Implement _should_block(pattern, tool_name, tool_input) -> bool:
       - blind_edits: tool_name in ("Edit", "Write", "NotebookEdit")
       - bash_failures: tool_name == "Bash"
       - thrashing: tool_name in ("Edit", "Write", "NotebookEdit") and tool_input.get("file_path", "") matches pattern.data.get("file", "")

    6. Implement _format_block(tool, target, kind, detail, suggestion, pressure) -> str:
       Per D-16 format: "[SOMA BLOCKED] {tool} on {target}\nReason: {detail}\nFix: {suggestion}\nPressure: {p:.0%}"

    7. Implement _target_name(tool_input) -> str:
       Extract meaningful target from tool_input: file_path for Edit/Write, command[:60] for Bash, "" otherwise

    Create tests/test_reflexes.py with test classes:
    - TestReflexResult: frozen dataclass, default values
    - TestBlindEditsReflex: blocks Edit without Read, allows Read, allows with toggle off
    - TestRetryDedup: blocks identical Bash, allows different command, whitespace normalization
    - TestBashFailuresReflex: blocks after 3 consecutive errors, allows non-Bash
    - TestThrashingReflex: blocks after 3 edits same file, allows different file
    - TestErrorRateReflex: inject only (allow=True, inject_message set)
    - TestInjectionOnlyReflexes: research_stall and agent_spam -> inject only
    - TestBlockFormat: message matches D-16 format
    - TestReflexConfig: per-reflex toggle, override_allowed
    - TestNoPatterns: clean action log -> allow=True

    Test helpers: use the same action_log format as patterns.py: list of dicts with keys {tool, error, file, ts}. Build action logs that trigger specific patterns (e.g., 3 Edit actions without Read for blind_edits).

    Create `src/soma/signal_reflexes.py` as a pure-function module (per D-03, separate from reflexes.py). Import ReflexResult from soma.reflexes.

    Implement 6 functions:

    1. `evaluate_predictor_checkpoint(prediction, soma_mode)` -> ReflexResult
       - Per D-01: fires when prediction.confidence > 0.7 AND prediction.actions_ahead <= 3
       - Per D-02: mode="reflex" -> reflex_kind="predictor_checkpoint" with checkpoint message. mode="guide" -> reflex_kind="predictor_warning" with warning only.
       - prediction can be None (no prediction available) -> allow=True, no reflex

    2. `evaluate_drift_guardian(drift, original_task, current_activity)` -> ReflexResult
       - Per D-04: fires when drift > 0.4
       - Per D-05/D-06: inject_message format "Your task: {original_task}. You're now doing: {current_activity}. Refocus."
       - If original_task is empty/None -> skip injection (graceful degradation per pitfall 3)

    3. `evaluate_handoff(success_rate, handoff_text, agent_id)` -> ReflexResult
       - Per D-07: fires when success_rate < 0.4
       - Per D-08: inject_message includes the handoff_text (already generated by halflife.py)
       - Returns trust_reduction field in detail for hook to apply (per D-09)

    4. `evaluate_rca_injection(error_rate, rca_text)` -> ReflexResult
       - Per D-10: fires when error_rate > 0.3
       - Per D-11: inject_message format "[SOMA DIAGNOSIS] Root cause: {rca_text}"
       - Per D-12: always allow=True (injection only, never blocks)
       - If rca_text is None -> skip injection

    5. `evaluate_commit_gate(grade, tool_name, tool_input)` -> ReflexResult
       - Per D-13: grade D/F -> allow=False with block_message
       - Per D-14: grade C -> allow=True with inject_message warning
       - Per D-15: only applies when tool_name="Bash" and command matches git commit regex `r"^\s*git\s+commit\b"` (per pitfall 2)
       - Non-commit tool calls -> allow=True, no reflex

    6. `evaluate_all_signals(vitals, pressure, action_log, agent_id, soma_mode, prediction, success_rate, handoff_text, original_task, current_activity, error_rate, rca_text)` -> list[ReflexResult]
       - Calls each evaluator, filters to non-empty results
       - Caps to max 2 injections, priority: rca > drift > handoff > checkpoint (per pitfall 4, commit gate handled separately in pre_tool_use)

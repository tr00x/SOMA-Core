**src/soma/benchmark/report.py** — Two output modes per D-04, D-17, D-18, D-19:

`generate_markdown(result: BenchmarkResult) -> str`:
- Returns complete Markdown string for docs/BENCHMARK.md
- Header: `# SOMA Benchmark Results`
- Metadata: timestamp, runs per scenario
- Summary table (for README per D-18):
  ```
  | Metric | Without SOMA | With SOMA | Improvement |
  |--------|-------------|-----------|-------------|
  | Errors | X.X% | Y.Y% | -ZZ% |
  | Retries | X.X% | Y.Y% | -ZZ% |
  | Tokens | XXXXX | YYYYY | -ZZ% |
  ```
- Per-scenario sections with:
  - Scenario name and description
  - Individual metrics table (error_reduction, retry_reduction, token_savings)
  - Mode transition summary
- Footer: `*Auto-generated by \`soma benchmark\`. Do not edit manually.*`

`render_terminal(result: BenchmarkResult) -> None`:
- Uses `rich.console.Console()`, `rich.table.Table`, `rich.panel.Panel`, `rich.progress.Progress`
- Per D-19: shows live progress during benchmark is NOT here (that's in CLI handler)
- Renders final results: overall summary panel, per-scenario tables
- Color coding: green for improvements > 10%, yellow for < 10%, red for regressions
- Shows error_reduction, retry_reduction, token_savings per scenario
- Shows overall summary at bottom

`render_progress(console, scenario_name, run_number, total_runs) -> None`:
- Simple progress line for live output during benchmark: `[scenario_name] Run {n}/{total}...`

**src/soma/cli/main.py** — Add `soma benchmark` subcommand per D-07:

In `_build_parser()`, add after the last subparser:
```python
benchmark_parser = subparsers.add_parser("benchmark", help="Run SOMA behavioral benchmarks")
benchmark_parser.add_argument("--scenarios", nargs="*", default=None, help="Specific scenarios to run (default: all)")
benchmark_parser.add_argument("--runs", type=int, default=5, help="Runs per scenario (default: 5)")
benchmark_parser.add_argument("--output", type=str, default="docs/BENCHMARK.md", help="Markdown output path")
benchmark_parser.add_argument("--json", type=str, default=None, help="JSON output path")
benchmark_parser.add_argument("--no-terminal", action="store_true", help="Skip rich terminal output")
benchmark_parser.add_argument("--tune-thresholds", action="store_true", help="Run threshold tuner on results")
```

Add `_cmd_benchmark(args)` handler:
- Import `from soma.benchmark import run_benchmark` and `from soma.benchmark.report import generate_markdown, render_terminal`
- Call `result = run_benchmark(runs_per_scenario=args.runs)`
- If not `args.no_terminal`: call `render_terminal(result)`
- Write `generate_markdown(result)` to `args.output` (create parent dirs)
- If `args.json`: write `json.dumps(asdict(result), indent=2)` to path per D-04
- If `args.tune_thresholds`: import and call `compute_optimal_thresholds`, print results
- Add `dispatch["benchmark"] = _cmd_benchmark`

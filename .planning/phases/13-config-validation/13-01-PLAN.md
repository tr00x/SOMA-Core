5. `context_exhaustion()` — 100 actions with token_count increasing from 100 to 5000 per action. Tests budget pressure and context usage signals.

Create `src/soma/benchmark/harness.py`:

**`run_scenario(actions, soma_enabled, agent_id="benchmark-agent", budget=None) -> BenchmarkMetrics`:**
- Creates fresh `SOMAEngine(budget=budget or {"tokens": 500_000})`
- Registers agent with `engine.register_agent(agent_id)`
- Iterates actions. If `soma_enabled` and `action.guidance_responsive`:
  - Check `engine.get_level(agent_id)` — if `>= ResponseMode.GUIDE`, skip action, record as guidance_followed
- For each processed action: call `engine.record_action(agent_id, Action(tool_name=..., output_text=..., token_count=..., error=..., retried=...))`
- Collect ActionMetric per action from ActionResult: `result.pressure`, `result.vitals.uncertainty`, `result.vitals.drift`, `result.vitals.error_rate`, `result.vitals.token_usage`, `result.vitals.cost`, `result.mode.name`, whether guidance was issued (`result.mode >= ResponseMode.GUIDE`), whether followed
- Track mode transitions: when `result.mode != previous_mode`, record `{from: prev.name, to: current.name, at_action: index, pressure: result.pressure}`
- After all actions: compute error_rate = total_errors / total_actions, retry_rate = total_retries / total_actions
- Compute false_positives/true_positives: for each action where mode >= GUIDE, check if any of next 3 actions had error=True. If yes: true_positive. If no: false_positive.
- Return BenchmarkMetrics with all collected data

**`run_multi_agent_scenario(agent_a_actions, agent_b_actions, soma_enabled, budget=None) -> tuple[BenchmarkMetrics, BenchmarkMetrics]`:**
- Creates single SOMAEngine, registers "agent-a" and "agent-b"
- Adds trust edge: `engine._graph.add_edge("agent-a", "agent-b", trust=0.8)`
- Interleaves actions: process agent_a[0], agent_b[0], agent_a[1], agent_b[1], ...
- Returns separate BenchmarkMetrics for each agent

**`run_benchmark(runs_per_scenario=5) -> BenchmarkResult`:**
- Runs all 5 scenarios from scenarios.py
- For each scenario: runs `runs_per_scenario` times with soma_enabled=True (seeds 1..N) and `runs_per_scenario` times with soma_enabled=False (same seeds)
- For multi_agent_coordination: calls run_multi_agent_scenario, uses agent-b metrics for comparison
- Computes per-scenario ScenarioResult with error_reduction, retry_reduction, token_savings
- Computes overall averages across all scenarios
- Returns BenchmarkResult with timestamp from `datetime.datetime.now().isoformat()`

Create `tests/test_benchmark.py` with tests for all behaviors listed above. Use small action lists (5-10 actions) in unit tests for speed. Include one integration test that calls `run_benchmark(runs_per_scenario=2)` with reduced scenario sizes.
